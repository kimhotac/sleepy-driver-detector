• 동공(검은 원형)
• 다양한 밝기 분포
• 복잡한 경계선
• 높은 밝기 분산 		• 눈꺼풀(평면)

• 강한 가로선
• 낮은 밝기 분산

🎯 접근 방법: 이 차이점들을 OpenCV로 직접 측정하자!

1. 전처리

컬러→흑백 → 명암개선 → 노이즈제거
⬇️

2. 특징 검출

동공검출 + 에지분석 + 분산계산
⬇️

3. 판정 및 융합

점수계산 → 가중융합 → 최종결과

⭐ 핵심 전략: 계층적 판정 (확실한 것부터 우선)

🔧 1단계: 전처리

Step 1: 흑백 변환

cv2.cvtColor(eye_img, cv2.COLOR_BGR2GRAY)

목적: 조명/피부색 노이즈 제거, OpenCV 함수 최적화

Step 2: 명암 대비 향상 (CLAHE)

cv2.createCLAHE(clipLimit=2.0, tileGridSize=(8,8))

매개변수 선택:

  • clipLimit=2.0: 과도한 대비 방지 (1.0~4.0 범위에서 중간값)
  • tileGridSize=(8,8): 90x90 이미지에 적합한 지역 크기

Step 3: 노이즈 제거 (가우시안 블러)

cv2.GaussianBlur(img, (5,5), 0)

(5,5) 선택 이유: 피부 질감 제거하면서 주요 경계 보존

  • (3,3)은 너무 약함 / (9,9)는 중요한 경계까지 흐려짐

🎯 2단계: 특징 검출

🥇 1순위: 동공 검출 (절대적 기준)

cv2.HoughCircles(..., minRadius=7, maxRadius=20)
논리: 동공이 보이면 100% 눈을 뜬 상태 → 즉시 0.0 반환
의의: False Positive 방지, 확실한 판정 기준 제공

🥈 2순위: 가로 에지 분석 (70% 가중치)

cv2.Sobel(img, CV_64F, 0, 1, ksize=5)
논리: 눈 감음 → 강한 가로선 (위아래 눈꺼풀 경계)
매개변수: ksize=5로 두꺼운 눈꺼풀 경계선까지 감지

🥉 3순위: 밝기 분산 (30% 가중치)

np.var(gray_eye)
논리: 눈 감음 → 균일한 밝기 → 낮은 분산
역할: 에지 검출의 보조 지표, 특수 상황 대응

⚙️ 핵심 매개변수 분석

함수 매개변수 선택 값 선택 근거
HoughCircles dp 1.0 최고 정밀도 (속도보다 정확도 우선)
param2 30 False Positive 방지 (너무 낮으면 노이즈)
Sobel ksize 5 두꺼운 눈꺼풀 경계선 감지 (3은 너무 얇음)
임계값 EDGE_MIN/MAX 5.0 / 30.0 경험적 눈 뜸/감음 범위
임계값 VARIANCE_MIN/MAX 300.0 / 1500.0 밝기 분산 기준값

📊 임계값 설정 근거

눈 뜸 ←── 5.0 ────── 30.0 ──→ 눈 감음

(에지 밀도 기준)


높은 분산(1500+) ←── 300 ──→ 낮은 분산

(밝기 분산 기준)

⚖️ 3단계: 판정 및 융합

IF 동공 검출됨:

→ 즉시 return 0 (확실히 눈 뜸)

의의: 절대적 기준으로 즉시 판정

ELSE:

final_score = 0.7 × edge_score + 0.3 × variance_score

가중치 근거:

  • 에지 0.7: 눈 감김의 가장 강력한 신호 (주요 특징)
  • 분산 0.3: 보조적 검증 지표 (환경 변화 대응)
if (result ≤ 0.95): return 0 # 눈 뜸
else: return 1 # 눈 감음

0.95 기준: 엄격한 판정으로 False Positive 방지

(확실하지 않으면 "눈 뜸"으로 보수적 판정)

⚠️ 솔직한 결과 분석

🔍 실패 원인

  • 개인차 무시: 눈꺼풀 두께, 속눈썹, 피부색 차이
  • 단순한 특징: 가로선+분산만으로는 복잡한 상황 대응 한계
  • 고정 임계값: 5.0~30.0이 모든 사람에게 맞지 않음
  • 시간적 정보 부족: 단일 프레임 분석의 한계

🔧 트러블슈팅: 매개변수 튜닝의 현실

😰 겪었던 문제들

  • 매개변수 기준 부재: clipLimit, ksize, 임계값을 어떻게 정해야 할지 막막
  • 무작정 시행착오: 1씩, 2씩 증가/감소하며 일일이 테스트
  • 변화 미미: 값을 바꿔도 결과에 큰 차이가 없어서 방향을 잡기 어려움
  • 객관적 평가 부족: "더 좋아졌다/나빠졌다"를 눈으로만 판단

💡 팀원 피드백 & 깨달음
